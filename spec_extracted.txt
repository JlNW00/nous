1. Executive Summary
This system is an autonomous on-chain intelligence agent — not a tool, not a dashboard, not a trading bot. It
continuously monitors Solana token launches, investigates them without human input, scores them using
deterministic signals plus LLM reasoning, and publishes public verdicts.
The original spec described this capability. This version reorders build priorities and adds Bags-native
integrations to qualify for and win the Bags $4M Hackathon.
What Makes This an Agent (Not a Tool)
 Reactive tools wait for input. This agent monitors, decides, acts, and posts autonomously.
 The discovery layer continuously polls Bags API for new launches and Helius for mint events.
 The agent has its own onchain wallet that receives partner fee shares.
 Verdicts are published publicly via dashboard feed and X/Twitter without human intervention.
 The agent re-investigates tokens on price spikes and volume anomalies.
Core Output
A credibility report with evidence, weighted scoring, confidence levels, and a verdict: Legitimate / Suspicious /
High Risk / Larp. Reports are versioned, auditable, and traceable to stored evidence.
Long-Term Moat
Entity resolution and capital lineage graphs. The same operators launch multiple projects from fresh wallets.
This system clusters actors, not just tokens — making it uniquely resistant to evasion over time.


2. Product Thesis
Most crypto scanners analyze isolated tokens. That is weak because the same operators launch multiple
projects from fresh wallets. The stronger approach is actor-centric analysis.
Core Questions the System Answers
 Who likely controls this project?
 Has this capital appeared in prior launches?
 Does the technical footprint support the public narrative?
 Are social signals organic or coordinated?
 Was this token launched on Bags, and does its onchain performance support its story?
Why Bags Is the Right Partner
 Bags is the leading Solana token launchpad — our investigation pipeline directly serves their ecosystem.
 Legitimate projects investigated by us route to Bags via embedded CTAs, driving verified trading volume.
 Our partner key earns 25% of fees from tokens traded via our referrals — fully onchain and auditable.
 The agent's autonomous operation aligns with Bags' thesis: builders who ship, not presenters who pitch.

5. Major Services
5.1 Discovery Service (Celery discover_worker — NEW PRIORITY)
Previously reserved. Now the first build priority because it is what makes the system autonomous rather than
reactive.
 Polls Bags API every 5 minutes for new token launches.
 Listens to Helius webhooks for new mint events on Solana mainnet.
 Monitors trending token feeds and social mentions as secondary sources.
 Creates a Case record and enqueues enrichment jobs automatically.
 Deduplicates by contract address, symbol, project name, domain, and known aliases.
 Assigns investigation priority using: volume velocity, Bags-launched flag, social spike detection.
5.2 Data Collection Service
Separates provider adapters from normalization logic. Stores all raw provider responses for auditability.
 Helius: token metadata, top holders, deployer identification, recursive funding trace.
 DexScreener: pairs, liquidity, volume, price, market creation time.
 GitHub: repo age, commit velocity, contributor count, fork count.
 Infrastructure: DNS records, WHOIS, SSL cert, HTTP probe for backend presence.
 Bags API (NEW): token launch data, lifetime fees, trading volume, partner stats, fee share config.
 Caches expensive calls (holder lists, funding traces) with TTLs per object type.
5.3 Entity Graph Engine
The system's primary moat. Shifts analysis from token-level metrics to actor-level relationships.
 Nodes: Wallet, Token, Contract, Entity, Exchange, Bridge, Domain, Repo, SocialAccount, Project.
 Edges: DEPLOYED, FUNDED_BY, TRANSFERRED_TO, CONTROLS, PROVIDES_LIQUIDITY_FOR,
LINKS_TO, MENTIONS.
 Clusters wallets into operator entities using shared funding, gas payer patterns, bridge routes, deployment
signatures.
 Supports traversal queries: capital origin paths, related launches, deployer families, shared infrastructure.
5.4 Investigation Engine
 Converts raw evidence into structured signals with confidence scores and evidence pointers.
 Builds a chronological timeline: funding → deployment → listing → social creation → code events.
 Produces a single structured investigation payload for the reasoning layer.
5.5 Reasoning & Scoring Layer
 Deterministic heuristics run first. LLM reasoning runs on top of structured evidence — never raw text dumps.
 Anthropic SDK (already integrated): receives structured JSON facts, returns verdict narrative.


 Required output fields: summary, supporting_findings, contradictions, open_questions, verdict_suggestion,
confidence.
 LLM cannot invent missing evidence. Missing data stays explicitly missing in output.
 Every prompt and response version stored and tied to the report for auditability.
 Fallback to deterministic-only mode if LLM unavailable.
5.6 Delivery Layer
 REST API: all endpoints live and functional.
 Dashboard: Next.js — search, report view, score breakdown, graph panel, live agent feed.
 Public feed: auto-investigated tokens appear in real-time as agent processes them.
 X/Twitter poster: agent auto-posts verdict summaries with confidence thresholds as safety gate.
 Historical versions: score drift and case evolution tracked over time.

11. Autonomous Agent Workflow
The key architectural change from v1: the pipeline is triggered autonomously by the discovery layer, not by
human input.
11.1 Autonomous Discovery Loop
Every 5 min: Poll Bags API for new launches
→ Filter: volume velocity > threshold OR Bags-launched
→ Deduplicate by contract/symbol/domain
→ Create Case record
→ Enqueue: fetch_worker, graph_worker, signal_worker
Real-time: Helius webhook → new mint event
→ Same pipeline as above, higher priority
11.2 Re-Investigation Triggers
 Price spike > 50% in 1 hour → re-investigate at higher confidence threshold.
 Volume anomaly: 10x average volume → re-investigate.
 Manual analyst submission via POST /cases/analyze.
 Scheduled: all active cases re-investigated every 24 hours.

12. LLM Reasoning Specification
The LLM operates on structured evidence, not raw scraped text. This is critical for auditability and preventing
hallucination.
Input Contract
 Structured JSON facts only — no uncontrolled raw dumps.
 Prompt frames the model as an investigation assistant, not a market predictor.
 All missing data explicitly marked as MISSING — model cannot invent evidence.

20. Security, Cost Controls & Risks
Security
 Sanitize and validate all external input identifiers before processing.
 Rate-limit public endpoints. Auth required for expensive operations.
 Separate provider secrets from app containers using environment secrets.
 Poster agent has confidence threshold gate — no unverifiable allegations posted automatically.
 All manual analyst overrides logged with provenance.
Cost Controls
 Cache provider results aggressively with TTLs based on object type.
 Run cheap deterministic checks before expensive recursive traces.
 LLM reasoning only invoked after deterministic enrichment is complete.
 Historical snapshots avoid re-pulling identical evidence on reruns.
 Batch social and chain requests where provider limits allow.
 Bags API: 1,000 req/hour limit — implement exponential backoff.
